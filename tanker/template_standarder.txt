Jeg synes f.eks. news_detail.html er gode navn på templates, og vi burde
konvertere det meste til samme system. Større bruk av class based generic
templates hvor det er mulig. Det fører til bl.a. enklere urls i views. Altså 
{% url news_detail %} fremfor {% url news.views.detail %} eller det som verre
er.

I tillegg bør reverse urls brukes der det er mulig. Bruk altså {% url
event_detail event.id %} fremfor /arrangement/{{ event.id }}. Dette gjør det
enormt mye lettere å endre url til f.eks. /events/ senere.
